Stageproject

Test Automation Framework voor QA Kamp

Bij Brightest een Belgisch IT-consultancybedrijf gespecialiseerd in softwaretesting ontwikkelde ik een test automation framework voor een educatief computerspel: het QA Kamp. Dit spel leert kinderen van 8 tot 16 jaar de basisprincipes van software testing en cybersecurity aan de hand van interactieve minigames.

Ik werkte in Java met tools zoals Playwright en Selenium om gebruikersinteracties automatisch te testen. Via GitHub en een duidelijke branchingstrategie zorgde ik voor gestructureerd versiebeheer. Het framework maakt regressietesten mogelijk bij verdere uitbreiding van het spel.

Bachelorproef · Academisch onderzoek

Ontwerp van een Test Automation Framework voor Interactieve Games

Mijn bachelorproef sloot rechtstreeks aan bij de stageopdracht. De centrale onderzoeksvraag luidde: "Hoe kan een schaalbaar en stabiel test automation framework ontworpen worden voor het automatisch testen van interactieve, UI-gedreven minigames?"

Ik onderzocht welke testsoorten (unit, integratie, end-to-end, UI) relevant zijn voor een gameplatform, vergeleek frameworks zoals Playwright, Selenium en Appium, en analyseerde uitdagingen zoals animaties, timing, randomness en flaky tests.

September - december 2025 Multidisciplinair project

IT-project: AI Predictiemodel

In een multidisciplinair team bouwden we een AI-predictiemodel van A tot Z. Van datavoorbereiding en modelselectie tot training, testen en iteratie. Dit project leerde mij hoe complex en tijdrovend het bouwen van een betrouwbaar model werkelijk is. Samenwerking met een echte klant en dagelijkse stand-ups maakten dit een intensieve maar waardevolle ervaring.

AI / Machine Learning Agile / Scrum Data cleaning Klantcommunicatie Multidisciplinair

Dit project leerde mij enorm veel en hielp mij groeien op zowel technisch als persoonlijk vlak. Werken met mensen uit verschillende studierichtingen toonde mij hoe krachtig een multidisciplinair team kan zijn wanneer ieders sterktes serieus worden genomen.

In het begin zorgden onze verschillende manieren van denken en werken soms voor wrijving. Na verloop van tijd werden die verschillen net een troef: elk van ons benaderde het probleem vanuit een unieke invalshoek, wat het eindresultaat versterkte.

Ik leerde ook wat je realistisch van een klant kan verwachten en hoe belangrijk het is om gezonde grenzen te stellen. Eerlijk zijn over de scope en alternatieven aanbieden houdt de samenwerking constructief en voorkomt frustratie aan beide kanten.

Het was uitdagend dat mijn specialisatie (Software Management) niet altijd even goed begrepen of gewaardeerd werd. Dat pustte mij om mijn toegevoegde waarde duidelijker te verwoorden en meer vertrouwen te winnen in mijn eigen expertise.

Uiteindelijk verstevigde dit project mijn technische basis, mijn communicatieve vaardigheden en mijn ervaring met Agile werken. Zijn ervaringen die ik meeneem naar elke volgende uitdaging.
